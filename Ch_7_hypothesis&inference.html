def inverse_normal_cdf(p, mu=0, sigma=1, tolerance=0.00001):
    """find approximate inverse using binary search"""
    # if not standard, compute standard and rescale
    if mu != 0 or sigma != 1:
        return mu + sigma * inverse_normal_cdf(p, tolerance=tolerance)
    low_z, low_p = -10.0, 0 # normal_cdf(-10) is (very close to) 0
    hi_z, hi_p = 10.0, 1 # normal_cdf(10) is (very close to) 1
    while hi_z - low_z > tolerance:
        mid_z = (low_z + hi_z) / 2 # consider the midpoint
        mid_p = normal_cdf(mid_z) # and the cdf's value there
        if mid_p < p:
        # midpoint is still too low, search above it
            low_z, low_p = mid_z, mid_p
        elif mid_p > p:
        # midpoint is still too high, search below it
            hi_z, hi_p = mid_z, mid_p
        else:
            break
    return mid_z

def normal_approximation_to_binomial(n, p):
    """finds mu and sigma corresponding to a Binomial(n, p)"""
    mu = p * n
    sigma = sqrt(p * (1 - p) * n)
    return mu, sigma

def normal_probability_above(lo, mu=0, sigma=1):
    return 1 - normal_cdf(lo, mu, sigma)
# it's between if it's less than hi, but not less than lo
def normal_probability_between(lo, hi, mu=0, sigma=1):
    return normal_cdf(hi, mu, sigma) - normal_cdf(lo, mu, sigma)
# it's outside if it's not between
def normal_probability_outside(lo, hi, mu=0, sigma=1):
    return 1 - normal_probability_between(lo, hi, mu, sigma)

def normal_upper_bound(probability, mu=0, sigma=1):
    """returns the z for which P(Z <= z) = probability"""
    return inverse_normal_cdf(probability, mu, sigma)
def normal_lower_bound(probability, mu=0, sigma=1):
    """returns the z for which P(Z >= z) = probability"""
    return inverse_normal_cdf(1 - probability, mu, sigma)
def normal_two_sided_bounds(probability, mu=0, sigma=1):
    """returns the symmetric (about the mean) bounds
    that contain the specified probability"""
    tail_probability = (1 - probability) / 2
    # upper bound should have tail_probability above it
    upper_bound = normal_lower_bound(tail_probability, mu, sigma)
    # lower bound should have tail_probability below it
    lower_bound = normal_upper_bound(tail_probability, mu, sigma)
    return lower_bound, upper_bound

How should we interpret the increase of power? (0.89 -> 0.94) The null hypothesis ($H_0$) has now been modified from p = 0.5 to p <= 0.5. We will regard any value of p, the ratio of heads to 1,000 coin flips, less than 0.5 as acceptable. However, the likelihood of observing p larger than 0.5 will decrease as p grows closer to 1.

When $H_0$: p = 0.5 (equivalent to mu = 500), it is supposed that any value of p (don't be confused with p-value, another statistical concept) significantly smaller or larger than 0.5 is considered as 'unlikely' (Two-sided). The degree of 'significance' is set by '$\alpha$'.

The $\alpha$ can be thought of as our confidence over a hypothesis, or a hypothesized parameter value. It shows how much we are willing to admit that our hypothesis is true. In our example ($H_0$: p = 0.5), $\alpha$ = 0.05 means that we suppose that the ratio of the number of heads to 1,000 coin flips is 2.5% likely to exceed 0.531. In other words, the cumulated probability of observing values of p above the hypothesized mean (0.5) by 1.96 x sigma (0.5 / sqrt(1000)) or greater is 0.025. At the same time, the ratio is 2.5% likely to go below 0.469. Similarly, the cumulated probability of observing values of p below the hypothesized mean by the same amount or greater, once $H_0$ is true, would be 0.025. The sum of the cumulated probabilities equals 0.05, $\alpha$. This 0.05 is equal to the sum of the area under pdf at (-inf, hypothesized mean - 1.96 x sigma) and (hypothesized mean + 1.96 x sigma, inf), each of which is 0.025.

Thus, we are going to reject $H_0$ if a value of p is observed outside the range (0.469, 0.531) among a numerous number of 1,000 coin flips. Why would we hold onto a hypothesis that is not likely to predict what we have found? Wouldn't it be better to find another hypothesis that is more likely to accomodate what we have observed?

In contrast, when $H_0$: p <= 0.5, observing a value smaller than 0.5 does not matter to the level of $\alpha$. Only a value of p larger than 0.5 is of interest (One-sided). Therefore, 5% of $\alpha$ level is equal to the area under pdf at (hypothesized mean + 1.64 x sigma, inf). The upper bound of 'one-sided' should, consequently, be smaller than that of 'two-sided'.

Now, $\beta$ is not something we could decide. It highly depends on the true value of p. In our example, it could be understood like this:

Suppose the p is not 0.5. Therefore, we bring an alternative hypothesis, $H_1$: p = 0.55. On the above pdf plot, this alternative pdf is the red curve. The thing is, we never know what the true p is. $H_1$ could be p = 0.53, p = 51, or whatever you may think of. The condition for $H_1$ is p other than 0.5.

Let's say that we just stick to our null hypothesis first.

Under $H_0$: p = 0.5 and $\alpha$ = 0.05, we don't reject that hypothesis when a value of p is within the range (0.469, 0.531). This range can also be observed with the red pdf, since every pdf's range is (-inf, inf). Now, with the alternative hypothesis ($H_1$), how much are we likely to observe a value of p within this range? Calculating the normal probability between 0.469 and 0.531, with our true parameters (p = 0.55 / mu = 550, sigma = 15.73), will give us the answer. In this case, $\beta$ = 0.113, power = 1-$\beta$ = 0.887.

$\beta$ may be, therefore, the probability that we don't reject our null hypothesis, even though it is false. This being 'false' is represented by calculating the cumulative probability of our alternative hypothesis, $H_1$. Consequently, 'power' is the probability that we choose the alternative hypothesis given that it is 'true'.

Ok, let's get back to the difference of power between the one-sided test and the two-sided test.

If $H_0$: p <= 0.5 and $\alpha$ = 0.05, we don't reject that hypothesis when a value of p is within the range (-inf, 0.526). What's the probability of observing a value of p within this range now in a pdf of p = 0.55? It is given by the normal probability between -inf and 0.526. The result is $\beta$ = 0.063, power = 0.94.

A two-sided test considers either direction, and thus makes the area overlapped between the two pdf curves larger by pushing either the lower bound or the upper one of the null hypothesis toward the alternative hypothesis. The larger area overlapped, the more probability the alternative hypothesis should forsake even though it may be a true one.

On the contrary, a one-sided test deals with one direction, and makes the area overlapped smaller by pulling either the lower bound or the upper bound toward the null hypothesis.

Nonetheless, this also depends on which direction you do the one-sided test. Let's check if the true p is less than 0.5, and the same applies.

Now that we have observed few things regarding $\alpha$ and power, how can we increase statistical power?

See how wikepedia has summarized few factors that influence statistical power. https://en.wikipedia.org/wiki/Statistical_power#Factors_influencing_power

Statistical power may depend on a number of factors. Some factors may be particular to a specific testing situation, but at a minimum, power nearly always depends on the following three factors:

  1. the statistical significance criterion used in the test -> Setting $\alpha$ larger will pull the boundary of the null hypothesis toward the hypothesized mean. Then, statistical power will decrease.
  2. the magnitude of the effect of interest in the population(effect size) -> In our example, it would be the difference between the peaks of $H_0$ and $H_1$. Distance between the two should be greater to make statistical power increase.
  3. the sample size used to detect the effect -> Effects are more easily observed in larger sample. In our example, if n = 100 instead of 1000, then p = 0.5 will yield 50 heads out of 100 coin flips. p = 0.55 will yield 55 heads out of 100 coin flips. Let's see what's different.

    if x >= mu:

    """a normalizing constant so that the total probability is 1"""
    return gamma(alpha) * gamma(beta) / gamma(alpha + beta)
def beta_pdf(x, alpha, beta):
    if x < 0 or x > 1: # no weight outside of [0, 1]
        return 0
    return x ** (alpha - 1) * (1 - x) ** (beta - 1) / B(alpha, beta)

def binom_posterior_pdf(x, y, n, alpha, beta):

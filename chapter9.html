stdin and stdout¶

sys.argv[0] and sys.argv[1] of the "egrep.py" file are described below.

egrep.py searches an input a user types at the commandline in a designated text file(=sys.stdin). It does so line by line: that's why the "egrep.py" file has a line that says

$"for{\ }line{\ }in{\ }sys.stdin"$

sys.stdin is used for all interactive input.

If regex, the variable for the typed input, is in a line, then that line will be written to stdout.

The '|' is the pipe character, which means "use the output of the left command as the input of the right command." Therefore, the output of the egrep.py command(lines that contain regex) will be the input of the line_count.py command.

line_count.py command does nothing but count every line that is contained in sys.stdin and print the sum total.

stdin is used for all interactive input (including calls to input()); stdout is used for the output of print() and expression statements and for the prompts of input() from: https://docs.python.org/3/library/sys.html

def book_info(td):
    """given a BeautifulSoup <td> Tag representing a book,
    extract the book's details and return a dict."""
    
    title = td.find("div", "thumbheader").a.text
    by_author = td.find('div', 'AuthorName').text
    authors = [x.strip() for x in re.sub("^By ", "", by_author).split(",")]
    isbn_link = td.find("div", "thumbheader").a.get("href")
    isbn = re.match("/product/(.*)\.do", isbn_link).groups()[0]
    date = td.find("span", "directorydate").text.strip()
    
    return{
        "title": title,
        "authors": authors,
        "isbn": isbn,
        "date": date
    }
